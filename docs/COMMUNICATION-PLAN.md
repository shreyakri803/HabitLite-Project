## 1. Purpose
This plan defines how information will be communicated, documented, and shared throughout the HabitLite project.  
Clear communication ensures alignment, visibility, and accountability across all phases of the project.

## 2. Communication Objectives
- Provide consistent project visibility
- Track progress against milestones
- Ensure timely escalation of risks and issues
- Maintain transparent documentation for review

## 3. Communication Methods

### A. Project Status Reporting
**Type:** Weekly Status Report  
**Format:** PDF/PNG uploaded to `/status-reports` folder  
**Frequency:** Weekly (simulated timeline)  
**Owner:** Project Manager  
**Includes:**  
- RAG status  
- Achievements  
- Next steps  
- Issues  
- Risks  

### B. Daily Task Updates
**Type:** Micro-updates  
**Method:** Trello Board / GitHub Issues  
**Frequency:** Daily (simulated)  
**Owner:** Shreya  
**Details:**  
All task progress, comments, and blockers logged in Trello and GitHub Issues.

### C. Design & Planning Communication
**Type:** Diagram uploads  
**Method:** `/designs` folder in GitHub  
**Frequency:** As needed
